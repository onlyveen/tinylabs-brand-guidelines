# Tiny Labs Brand Guidelines

An interactive, self-contained brand identity style guide for Tiny Labs — covering logo usage, color palette, typography, and visual language.

## Overview

This repository contains the official Tiny Labs brand guidelines, delivered as a single-page interactive HTML document. It documents how to correctly apply the brand across all touchpoints.

## Contents

| File / Folder | Description |
|---|---|
| `style-guide.html` | Interactive brand guidelines (single HTML file, no dependencies) |
| `TinyLabs-Brand-Guide.pdf` | Printable PDF version of the brand guidelines |
| `Logo/` | Logo assets for light and dark backgrounds |
| `Type-Logo/` | Wordmark variants (Black, Solid, White) |
| `unacceptable-usage/` | Examples of incorrect logo usage |
| `visual-language.png` | Visual language reference |

**Typography:** Geist (body) · DynaPuff (display)

## Usage

Open `style-guide.html` directly in any modern browser — no build step or server required.

## Logo Guidelines

- Use the appropriate logo variant for the background (light vs. dark)
- Do not distort, recolor, add shadows, strokes, or visual effects to the logo
- Do not place the logo inside unapproved containers
- Refer to the `unacceptable-usage/` folder for visual examples of what to avoid
